# Milestones - Last Shift

## M0 - Repository Foundation

Goal: Prepare the project for development.

Target outputs:
- Unreal-style folder structure
- README
- Core docs
- Roadmap
- Issue backlog
- QA and release checklists

Exit criteria:
- Repository is clean and understandable on GitHub.
- Level 1 scope is clearly defined.

## M1 - Playable Blockout

Goal: Build the first walkable version of Level 1.

Target outputs:
- Start Office
- Main Corridor
- Storage Room
- Fuse Room
- Terminal Area
- Control Room

Exit criteria:
- Player can move through the complete planned route.
- Level scale and navigation feel reasonable.

## M2 - Core Gameplay Loop

Goal: Implement the minimum systems needed to complete Level 1.

Target outputs:
- Interaction prompt
- Interactable interface
- Door logic
- Fuse pickup
- Power restoration
- Terminal unlock
- Control room access

Exit criteria:
- Player can complete the loop from start to control room unlock.

## M3 - Story and Atmosphere Pass

Goal: Make the blockout feel like a horror experience.

Target outputs:
- Environmental storytelling props
- Warning notes
- Terminal messages
- First lighting pass
- Ambient audio and triggers

Exit criteria:
- The player understands the evacuation setup and feels rising tension.

## M4 - Lockdown Ending

Goal: Finish Level 1 with a clear scripted ending.

Target outputs:
- Final terminal interaction
- Lockdown state
- Alarm lighting
- Locking door feedback
- Final entity sound
- Fade or ending message

Exit criteria:
- Level 1 has a complete ending and does not rely on placeholder stopping points.

## M5 - Portfolio Build

Goal: Prepare the prototype for sharing.

Target outputs:
- QA pass
- Windows build
- Screenshots
- Gameplay video
- README polish
- Changelog update

Exit criteria:
- Project can be opened, played, built, and presented from GitHub.
